Flight explorer (advanced)

Scan cheapest fares from one origin to ~180 worldwide IATA airports by default (EU hubs plus Americas, Middle East, Asia, Oceania, Africa). Supports round trip (A/R) and multiple outbound days — the table shows the best price per destination across those days (via fli). Filters (non-EU, exclude UK, etc.) only narrow which of those airports are queried; Google may still return no fare for some pairs.

Quick — best round trip (A/R)

Pick origin and region. Uses the calendar search: start tomorrow, 6 months ahead, 7-night trip. EU = destination in the EU-27; International = outside the EU (e.g. UK, US, Turkey, Switzerland). Cheapest out/return dates only — no times.

Outbound window: max 14 days. A/R uses return = outbound + trip length. Schedule shows local first-leg departure → last-leg arrival per direction (from Google).

Automate: best A/R in a long horizon

Uses the calendar API (like fli dates --round): one search per destination covers the whole period (~few calls per 61-day chunk). You fix trip length; dates shown are the cheapest outbound/return pair found. Times-of-day are not returned by this API (use grid scan for schedules).

Hotels (optional)

After a round trip or calendar run, Xotelo adds a floor nightly rate (USD) per destination (TripAdvisor city → several listings, parallel /rates, best OTA). Flight and hotel currencies differ; totals are not merged. Official hotel star ratings are not in the API; “5★ class” uses TripAdvisor traveler rating ≥ 4.5.
